## Result

`curvelens-core` already contains every tracked pipeline, analytics, storage,
dashboard, agent, test, configuration, and WTI knowledge file in the original.
It also contains later product-profile and query-toolkit work absent from the
original repository.

The only committed WTI capability missing from the destination was the richer
Telegram message formatter in `agent/notify.py`. It adds:

- daily settle/IV changes and days to expiry
- curve and volatility term structure
- historical percentiles and 30-day range
- seasonal EIA context
- agreement streak and scenario-state board
- option OI walls/max-pain context
- next scheduled event and richer catalyst summary
- priority-alert triggers, extremes, scenario levels, and calibration scorecard

That formatter has been migrated with product-aware currencies, units,
benchmark labels, and optional fundamentals. The migration deliberately does
not copy the original's hard-coded USD/bbl, Brent-WTI, or EIA assumptions.

## Excluded from migration

- `.env`, credentials, runtime market data, manifests, reports, outbox state,
  caches, IDE files, and OpenClaw workspace-state files
- generated bytecode and test caches
- obsolete hard-coded product assumptions superseded by framework profiles

## Retirement sequence

1. Merge and deploy the parity change.
2. Run one WTI report through `notify.py --prepare` and compare its queued text
   with the original format.
3. Point the WTI agent/cron working directory at `curvelens-core`, preserving
   the intended WTI data directory and Telegram destination.
4. Confirm one daily brief and any event mini-run are delivered and acked once.
5. Disable the original repository's schedules before archiving it.

Do not delete the original checkout until the first replacement production run
has completed. Runtime data should be archived separately rather than committed
to this repository.
